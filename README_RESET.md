# RESET 명령어

**로컬 저장소***(매우 중요)* 의 변경사항을 되돌리기 위해 사용합니다.

- git reset --<설정> HEAD~<무를 수>
- git reset --<설정> HEAD^

후자는 한 개의 변경만 되돌리는 경우 대신 사용가능한 옵션입니다. (HEAD~1 과 같음)

설정:
# Soft: git commit 명령어를 치기 직전 상태로 되돌린다.
 - git log에 변경 사항이 "남아있지 않음"
 - 로컬 저장소에 변경 사항이 "남아있음"
 - staged 상태가 된 것은 "유지됨"
 - git commit 명령어를 쳐 즉시 재커밋할 수 있음
# Mixed(기본값): git add 명령어를 치기 직전 상태로 되돌린다.
 - git log에 변경 사항이 "남아있지 않음"
 - 로컬 저장소에 변경 사항이 "남아있음"
 - staged 상태가 된 것은 "되돌려짐 (unstaged)"
 - 재커밋을 위해 git add 부터 필요
# Hard(위험): 마지막 push 이후 변경 사항을 모두 삭제하고 애초에 없었던 것으로 처리
 - git log에 변경 사항이 "남아있지 않음"
 - 로컬 저장소에 변경 사항이 "남아있지 않음"
 - staged 상태가 된 것은 "되돌려짐 (완전 소멸)"
 - 일반적인 상황에서 복구 불가능

# 공통 주의사항
 - 위에서 썼듯이, 반.드.시. **로컬 저장소** 에.서.만. 사용해야 합니다.
 - 원격 저장소에서 이걸 썼다간 GIT이 꼬여 프로젝트 기여자들에게 많은 질타를 받게 될 것임
